# 仕様書 (Specification)

## 1. プロジェクトの目的
日々の業務タスクをチャット感覚で素早く記録し、半期ごとの成果面談の際に生成AI（ChatGPT等）で要約しやすい形式（Markdown）で保存するデスクトップアプリ。
UXの根幹は **「Fire & Forget（入力してすぐ離脱）」** にある。

## 2. 機能要件
- **タスク記録機能**: 任意のタグとテキストを日時付きで保存できる。
- **サジェスト機能（オートコンプリート）**: タグ入力時に、インクリメンタルなキーワード絞り込みによる予測・候補表示が行える。キーボードの上下カーソルキーとEnterのみで選択可能（マウス操作を排除）。
- **Markdownロギング**: 月別 (`YYYY-MM_log.md`) にファイルが分割されて保存される。

## 3. UI/UXの設計方針
- **CustomTkinter** を使ったシンプルかつモダンなダーク系UI。
- 起動時は自動的にタグの入力欄にフォーカスが当たる。
- アプリ内に過去の履歴を表示するエリアを持つ（現在は非アクティブに設定し「表示・追記専用」）。
- 入力を完了すると再読み込み等の処理を待つことなく**画面を即座に破棄（終了）する**。常駐することによるメモリ消費を防ぐ。

## 4. データ構造定義

### タグデータ
- `tags.json` に配列として保存される。
- `["アプリ作成", "会議", "ドキュメント整理"]` のようなシンプルな1次元配列。重複を許さない。

### ログデータ (Markdown)
`%USERPROFILE%\Documents\log\{YYYY-MM}_log.md`
生成AIが読み取りやすいよう、以下のような構造で保存される。

```markdown
## 2026-04-18 15:30
- [タグ1] [タグ2]
- ここにタスクの内容が入ります。改行も反映されます。

## 2026-04-18 17:45
- [ミーティング]
- 〇〇プロジェクトの進捗報告。
```
