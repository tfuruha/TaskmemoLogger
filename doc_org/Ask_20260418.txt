Listed directory tasklog

半期ごとの成果面談に向けて、日々のタスクをチャット感覚で記録し、後で生成AIに要約させるアプリですね。
とても実用的で素晴らしいアイデアだと思います！

仕様（実装計画）を固めるために、いくつか具体的な要件についてヒアリングさせてください。
以下の項目について、現在イメージしている内容やご希望を教えていただけますでしょうか。

### 1. 技術スタック（開発言語・フレームワーク）
現在ログインされているワークスペースのフォルダ名が `work_python` となっていますが、デスクトップアプリの開発には **Python** を使用する想定でしょうか？
（Pythonであれば、モダンなUIが作れる `CustomTkinter` や `Flet` 、あるいは `PyQt` などのライブラリが考えられます。もし他の言語がご希望であればお知らせください）
- A1: python + CustomTkinter


### 2. 保存するテキストファイルの形式
後日、生成AI（ChatGPTなど）に渡すことを考慮すると、AIが構造を理解しやすい形式にするのがおすすめです。以下のような形式のうち、どれがイメージに近いですか？
- **Markdown形式 (.md):** （例：日時の見出しの下に箇条書きで内容が続く。最もAIが読み取りやすく、人間も読みやすい）
- **プレーンテキスト (.txt):** （例：単に一行ずつ、または改行区切りで日時と内容が並ぶ）
- **JSON / CSV形式:** （より強い構造化データにしたい場合）

- A2: Markdown形式 (.md) 日時の下に箇条書きで内容が続く形式で

### 3. UIの機能要件（Teamsチャット風について）
Teamsのチャット画面のように運用するにあたり、以下の機能は必要でしょうか？
- **画面構成:** 「過去の記録がスクロールして見れる表示エリア」＋「画面下部に入力欄と送信ボタン」というイメージで合っていますか？
- A3-1: 過去の記録がスクロールして見れる表示エリア」＋「画面下部に入力欄と送信ボタン」で。
 過去の記録は今日分のみ表示でOK。それより前はファイル単位でテキストエディタで見ます。
- **修正・削除:** 一度送信（記録）したタスクの「編集」や「削除」ができる必要はありますか？（間違えて入力した場合など）
- A3-2: 修正・削除は不要。 
- **検索機能:** 過去に入力した内容をアプリ上でキーワード検索できる機能は必要ですか？
- A3-3: 検索機能は不要。 ただ、タグやキーワードの候補が出ると、表現の揺れが減ってAIが要約しやすくなるので、候補表示は欲しい。

### 4. 記録するデータの種類（カテゴリ・タグ付け）
タスクを入力する際、テキストだけでしょうか？
後でAIに「プロジェクトごとの割合」や「業務ジャンルごとの要約」を作らせるために、簡単な**カテゴリ選択（例：「開発」「会議」「サポート」等）**や**タグ付け**機能を含めますか？
― A4 カテゴリ タグがあったほうがAIが要約しやすいので、あったほうがいいです。
