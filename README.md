# TaskmemoLogger

デスクトップで素早くタスクの実績を記録し、後で生成AIに要約させるための  
Markdown ログを生成する「Fire & Forget 型」アプリケーションです。

## 特徴

- **爆速の UX**: `Ctrl+Enter` を押すだけで即座に保存してアプリ終了。次の作業にすぐ戻れます
- **始業時間記録機能**: 毎日の作業開始時に「始業」ボタンをワンクリックするだけで、特別な始業ログを記録してアプリが終了します（PCのローカル日付に基づく1日1回制限。前日以前の記録や深夜・早朝の起動でも正しく判定されます）
- **記録時刻修正機能**: 記録忘れなどに対応するため、プルダウンで記録時刻を「今 / -15分 / -30分 / -60分」から選択して遡及記録できます
- **チャット風 UI**: 直近20件のタスクがチャットバブル形式でスクロール表示され、追記感覚で入力できます（始業ログは緑色で視覚的に強調されます）
- **タグ pill 入力 + オートコンプリート**: タグを入力中に過去のタグが前方一致候補として表示されます。また、入力値が空の際や起動時のフォーカス時には `config.json` に設定した優先タグ候補（プロジェクト名など）がドロップダウン表示されます。キーボードのみで選択でき、選択済みタグは pill として表示されます
- **Markdown 出力**: 月ごとに `YYYY-MM_log.md` 形式で保存。ChatGPT / Claude などの AI にそのまま読み込ませて月次・半期レポートの要約が行えます
- **単一 EXE 配布**: Python ランタイム不要。`TaskmemoLogger.exe` 1ファイルで動作します

## 動作環境

| 項目 | 要件 |
|------|------|
| OS | Windows 10 (Build 19041以降) / Windows 11 |
| ランタイム | WebView2 (Windows 11 は標準搭載 / Win10 は自動インストール) |
| その他 | 追加インストール不要 |

## セットアップ

1. [GitHub Releases (最新版)](https://github.com/tfuruha/TaskmemoLogger/releases/latest) から `TaskmemoLogger.exe` をダウンロードします  
2. ダウンロードしたファイルを任意のフォルダに配置して実行します
3. 初回起動時、以下のファイル・フォルダが自動的に作成されます:
   - `%USERPROFILE%\Documents\TaskmemoLogger\config.json` — 設定ファイル（AI集計ルール等の定義）
   - `%USERPROFILE%\Documents\TaskmemoLogger\tags.json` — タグ候補データ
   - `%USERPROFILE%\Documents\TaskmemoLogger\log\` — ログ保存フォルダ（同一フォルダ配下に統合）

## 使い方

### 通常のタスク記録
```
起動
  ↓
タグ欄にフォーカス（自動）
  ↓
タグを入力（候補が出たら ↑↓ + Enter で選択 / そのまま Enter で新規追加）
  ↓
Tab キーでタスク内容欄へ移動
  ↓
タスク内容を入力
  ↓
（必要に応じて「記録時刻」プルダウンで遡及時間を選択）
  ↓
Ctrl+Enter （または「保存して閉じる」ボタン） → 保存して即終了
```

### 始業時間の記録
```
起動
  ↓
「始業」ボタンをクリック
  ↓
始業ログを記録して即終了
```

## キーボードショートカット

| キー | 動作 |
|------|------|
| `Tab` | タグ入力欄 → タスク内容欄へ移動 |
| `↑` / `↓` | タグ候補リストを移動 |
| `Enter`（候補表示中） | タグを選択して pill 化 |
| `Enter`（候補なし） | 入力中のタグを新規追加して pill 化 |
| `Backspace`（タグ欄・空） | 末尾の pill を削除 |
| `Escape` | 候補リストを閉じる |
| `Ctrl+Enter` | タスクを保存してアプリ終了 |

## 出力ファイルの形式

`%USERPROFILE%\Documents\TaskmemoLogger\log\YYYY-MM_log.md` に以下の形式で保存されます。新規月でファイルが作成される際に、自動的に先頭にYAML形式のメタデータとAI用の集計仕様が追加されます。

```markdown
---
target_month: 2026-07
rules:
  - "このファイルは2026年07月のタスクログです。"
  - "時刻はタスク終了を示すこと。"
  - "タスク開始は直前のタスクに紐づいている時刻（始業タグまたは前タスクの終了時刻）であること。"
  - "昼休みは12:00から13:00。"
  - "始業タグがない日は8:00始業とすること。"
  - "工数は15分単位（0.25時間単位）に丸めて集計すること。"
  - "「[休憩]」または「[私用]」タグが含まれるタスクは、工数集計の対象外とすること。"
summary_template: |
  以下のフォーマットで出力してください：
  ### 1. タグ別工数集計
  - [タグ名]: XX時間 (全体に対する割合%)
  ### 2. 主な業務成果の要約
  - [タグ名]: 業務の要約と成果
---

## 2026-07-01 09:00
- [始業]
- 始業

## 2026-07-01 15:30
- [会議] [プロジェクトA]
- 進捗報告MTG。次回アクションを確認。
```

## 設定ファイル (config.json)

`%USERPROFILE%\Documents\TaskmemoLogger\config.json` にて、新規ログファイル作成時に自動挿入される集計用ルールやテンプレートの指定、および起動時等に表示する優先タグ候補の定義が行えます。

```json
{
  "rules": [
    "このファイルは{year}年{month}月のタスクログです。",
    "時刻はタスク終了を示すこと。",
    "タスク開始は直前のタスクに紐づいている時刻（始業タグまたは前タスクの終了時刻）であること。",
    "昼休みは12:00から13:00。",
    "始業タグがない日は8:00始業とすること。",
    "工数は15分単位（0.25時間単位）に丸めて集計すること。",
    "「[休憩]」または「[私用]」タグが含まれるタスクは、工数集計の対象外とすること。"
  ],
  "summary_template": "以下のフォーマットで出力してください：\n### 1. タグ別工数集計\n- [タグ名]: XX時間 (全体に対する割合%)\n### 2. 主な業務成果の要約\n- [タグ名]: 業務の要約と成果",
  "priority_tags": [
    "プロジェクトA",
    "プロジェクトB"
  ]
}
```
* `{year}` と `{month}` は、ログファイル生成時にその年月に自動置換されます。
* `priority_tags` 配列に登録されたタグは、タグ入力欄が空の際や起動時のフォーカス時に優先的なサジェスト候補（プロジェクト候補タグなど）としてドロップダウン表示されます。

## タグ候補ファイル (tags.json)

`%USERPROFILE%\Documents\TaskmemoLogger\tags.json` にタグが蓄積されます。  
テキストエディタで直接編集できます（不要なタグの削除など）。

```json
[
  "会議",
  "開発",
  "ドキュメント",
  "レビュー"
]
```

## 開発・ビルド

```

Go 1.23+
Node.js 18+
Wails v2  (go install github.com/wailsapp/wails/v2/cmd/wails@latest)

# 開発モード（ホットリロード）
wails dev

# リリースビルド
wails build
# → build\bin\TaskmemoLogger.exe が生成されます
```
