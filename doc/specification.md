# 仕様書 (Specification)

## 1. プロジェクトの目的

日々の業務タスクをチャット感覚で素早く記録し、半期ごとの成果面談の際に  
生成 AI（ChatGPT / Claude 等）が要約しやすい Markdown 形式で保存するデスクトップアプリ。

UX の根幹は **「Fire & Forget（入力してすぐ離脱）」** にある。

## 2. 技術スタック

| 項目 | 内容 |
|------|------|
| フレームワーク | [Wails v2](https://wails.io/) |
| バックエンド | Go 1.23+ |
| フロントエンド | Vanilla HTML / CSS / JavaScript（フレームワークなし） |
| ビルドツール | Vite（フロントエンドバンドル）|
| 配布形式 | 単一 EXE（WebView2 埋め込み） |
| 対象 OS | Windows 10 (Build 19041+) / Windows 11 |

## 3. 機能要件

### 3.1 タスク記録
- タグ（複数可）とテキストを入力し、日時と共に Markdown ファイルへ追記する
- `Ctrl+Enter` で即座に保存してアプリを終了する（Fire & Forget）

### 3.2 チャット履歴表示
- 直近20件のタスクログをチャットバブル形式でスクロール表示する（当月・前月のログファイルを対象とする）
- 合計件数が20件未満の場合は、存在する分だけ表示する
- 履歴は読み取り専用とする（編集・削除機能は不要）
- 今日以外のエントリは `MM/DD HH:MM` 形式でタイムスタンプを表示する

### 3.3 タグ入力・オートコンプリート
- タグ入力中、入力前方一致（大文字・小文字区別なし）でタグ候補をドロップダウン表示する
- キーボード（`↑` `↓` `Enter`）のみで候補を選択できる（マウス不要）
- 選択されたタグは pill（錠剤型チップ）として表示され、`×` ボタンまたは `Backspace` で削除できる
- 候補に存在しない文字列をそのまま `Enter` することで新規タグとして登録できる

### 3.4 タグの永続化
- 使用したタグは自動的に `tags.json` へ保存される（重複なし）
- `tags.json` はテキストエディタで直接編集可能

### 3.5 Markdown ロギング
- 月別に `YYYY-MM_log.md` ファイルへ追記する
- 月が変わると自動的に新しいファイルへ書き込む
- フォーマットは Python 版と完全互換とする

### 3.6 シングルインスタンス制御
- アプリを二重起動した場合、2件目のプロセスは黙って終了する
- `tags.json` の同時書き込みによるデータ破壊を防ぐ目的

### 3.7 AI集計用YAMLヘッダの自動付与
- 毎月の初回ログ書き込み時（新規ログファイル生成時）、`config.json` から設定データを読み込み、ファイル先頭に YAML 形式のヘッダを自動的に書き出す
- ヘッダには、対象年月、AIに対する工数集計ルール（丸め、除外、終了時刻扱い等）、および推奨の集計テンプレート（フォーマット）を記載する
- ルールテンプレート内の `{year}` と `{month}` は自動で現在の年月に置換される
- すでにファイルが存在する場合は、ヘッダの追加は行わず追記処理のみを実行する

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| 起動速度 | タスクバー等からダブルクリック後、即座に入力できる状態になること |
| 常駐なし | 保存後にプロセスが終了し、メモリを解放すること |
| 配布 | Python ランタイムや追加インストールなしで動作する単一 EXE であること |
| データ互換 | 既存の `%USERPROFILE%\Documents\TaskmemoLogger\log\` への移行後も Python 版ログと同一フォーマットで共存できること |
| 時刻基準（タイムゾーン） | ログ記録、履歴表示、始業ガードなどのすべての時間判定は、PCのローカルタイムゾーンを基準として処理する。UTC変換等による深夜・早朝（0:00〜8:59など）の判定ズレを防ぐこと |

## 5. データ構造定義

### 5.1 タグデータ

**ファイルパス:** `%USERPROFILE%\Documents\TaskmemoLogger\tags.json`

```json
["会議", "開発", "ドキュメント", "レビュー"]
```

- UTF-8 エンコード
- 重複なし（大文字・小文字を区別しない）
- テキストエディタで直接編集可能な場所に保存する

### 5.2 ログデータ（Markdown）

**ファイルパス:** `%USERPROFILE%\Documents\TaskmemoLogger\log\YYYY-MM_log.md`

```markdown
---
target_month: 2026-07
rules:
  - "このファイルは2026年07月のタスクログです。"
  - "時刻はタスク終了を示すこと。"
  - "タスク開始は直前のタスクに紐づいている時刻（始業タグまたは前タスクの終了時刻）であること。"
  - "昼休みは12:00から13:00。"
  - "始業タグがない日は8:00始業とすること。"
  - "工数は15分単位（0.25時間単位）に丸めて集計すること。"
  - "「[休憩]」または「[私用]」タグが含まれるタスクは、工数集計の対象外とすること。"
summary_template: |
  以下のフォーマットで出力してください：
  ### 1. タグ別工数集計
  - [タグ名]: XX時間 (全体に対する割合%)
  ### 2. 主な業務成果の要約
  - [タグ名]: 業務の要約と成果
---

## 2026-07-01 09:00
- [始業]
- 始業

## 2026-07-01 15:30
- [会議] [プロジェクトA]
- ここにタスクの内容が入ります。

## 2026-07-01 17:45
- [開発]
- 進捗報告。
  2行目以降はスペース2つでインデント。
```

- 月ごとにファイルを分割する
- 新規ファイル生成時のみ、先頭に `---` で囲まれた YAML 形式のヘッダを自動挿入する
- タグなしで記録した場合、タグ行（`- [...]`）は省略する
- 複数行テキストは先頭行をそのまま、2行目以降をスペース2つでインデントして保存する

## 5.3 設定データ（JSON）

**ファイルパス:** `%USERPROFILE%\Documents\TaskmemoLogger\config.json`

```json
{
  "rules": [
    "このファイルは{year}年{month}月のタスクログです。",
    "時刻はタスク終了を示すこと。",
    "タスク開始は直前のタスクに紐づいている時刻（始業タグまたは前タスクの終了時刻）であること。",
    "昼休みは12:00から13:00。",
    "始業タグがない日は8:00始業とすること。",
    "工数は15分単位（0.25時間単位）に丸めて集計すること。",
    "「[休憩]」または「[私用]」タグが含まれるタスクは、工数集計の対象外とすること。"
  ],
  "summary_template": "以下のフォーマットで出力してください：\n### 1. タグ別工数集計\n- [タグ名]: XX時間 (全体に対する割合%)\n### 2. 主な業務成果の要約\n- [タグ名]: 業務の要約と成果"
}
```

- UTF-8 エンコード。存在しない状態でアプリを起動した際にデフォルトの内容で自動生成される。
- `rules` 配列内に記述された各文言の `{year}`, `{month}` は、ログ出力時に該当する年月に置換される。

## 6. UI/UX 設計方針

- ダークテーマ（`#1a1b1e` ベース）を採用し、長時間使用しても目に優しい配色とする
- 起動直後にタグ入力欄へ自動フォーカスし、即座にキーボード入力を受け付ける
- ウィンドウサイズ: 通常 520 × 630 px（最小 400 × 460 px）
- ネイティブタイトルバーを使用する（Frameless なし）
- 修正・削除・検索機能は持たない（シンプルさを優先）

## 7. キーボードショートカット仕様

| キー | 動作 |
|------|------|
| `Tab` | タグ入力欄 → タスク内容欄へフォーカス移動 |
| `↑` / `↓` | サジェストリスト内を移動 |
| `Enter`（候補選択中） | タグを確定して pill 化 |
| `Enter`（候補なし） | 入力中のテキストを新規タグとして追加 |
| `Backspace`（タグ欄・空） | 末尾の pill を削除 |
| `Escape` | 候補リストを閉じる |
| `Ctrl+Enter` | タスクを保存してアプリ終了 |
